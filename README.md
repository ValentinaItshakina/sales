## Инструкия по сборке
```
cmake_minimum_required(VERSION 3.10)

project(SALES VERSION 1.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

set(SOURCES
    src/main.cpp
    src/Product.cpp
    src/Warehouse.cpp
)

add_executable(${PROJECT_NAME} ${SOURCES})

target_include_directories(${PROJECT_NAME} PUBLIC include)

if(MSVC)
    target_compile_options(${PROJECT_NAME} PRIVATE /W4)
else()
    target_compile_options(${PROJECT_NAME} PRIVATE -Wall -Wextra)
endif()
```


## Описание классов и их методов
### Класс Product

#### Описание
Класс представляет товар на складе с уникальным идентификатором, названием, ценой и количеством.

#### Поля
| Модификатор | Тип | Название | Описание |
|-------------|-----|----------|----------|
| private | std::string | productId | Уникальный идентификатор товара |
| private | std::string | name | Название товара |
| private | double | price | Цена товара |
| private | int | quantityInStock | Текущее количество на складе |

#### Конструктор
| Сигнатура | Описание |
|-----------|----------|
| `Product(std::string id, std::string n, double p, int q)` | Создает товар с указанными параметрами |

#### Методы
| Модификатор | Возвращаемый тип | Сигнатура | Описание |
|-------------|------------------|-----------|----------|
| public | void | `restock(int quantity)` | Увеличивает количество товара на складе |
| public | void | `sell(int quantity)` | Уменьшает количество товара при продаже |
| public | bool | `isAvailable() const` | Проверяет наличие товара (quantity > 0) |
| public | std::string | `getId() const` | Возвращает идентификатор товара |
| public | std::string | `getName() const` | Возвращает название товара |
| public | double | `getPrice() const` | Возвращает цену товара |
| public | int | `getQuantity() const` | Возвращает количество на складе |

### Класс Warehouse

#### Описание
Класс управляет списком товаров на складе, предоставляя функции для добавления, поиска и формирования отчетов.

#### Поле
| Модификатор | Тип | Название | Описание |
|-------------|-----|----------|----------|
| private | `std::vector<Product>` | catalog | Вектор для хранения товаров на складе |

#### Конструктор
| Сигнатура | Описание |
|-----------|----------|
| `Warehouse() = default` | Конструктор по умолчанию (создает пустой склад) |

#### Методы
| Модификатор | Возвращаемый тип | Сигнатура | Описание |
|-------------|------------------|-----------|----------|
| public | void | `addProduct(const Product& product)` | Добавляет товар в каталог склада |
| public | Product* | `findProduct(const std::string& productId)` | Ищет товар по ID. Возвращает указатель на товар или `nullptr`, если товар не найден |
| public | void | `generateInventoryReport() const` | Выводит отчет о всех товарах на складе |

### Проблема с архитектурой взаимодействия классов

Изначально реализована архитектура, при которой все операции с продуктами (создание, изменение количества) осуществлялись исключительно через методы склада. Такой подход приводил к необходимости написания дополнительных вспомогательных функций и усложнял логику программы.
Решение проблемы с архитектурой взаимодействия классов

**Выбрано решение:** Провести анализ предметной области складской логистики, в ходе которого были выделены ключевые принципы:
Продукт как самостоятельная сущность — товар существует независимо от склада, имеет собственное состояние (количество, цена, характеристики)
Склад как система хранения — отвечает за учёт и агрегацию товаров, но не управляет их жизненным циклом
Чёткое разделение ответственности — продукт управляет своими операциями (sell, restock), склад — каталогизацией и отчётностью
Выбрано решение: Реализовать архитектуру с разделением ответственности: сначала создаются экземпляры класса Product с начальным количеством, затем они добавляются на склад через addProduct(). Все последующие операции с количеством (продажи, пополнение) выполняются напрямую над объектами Product, найденными через findProduct(). Склад не создаёт новые продукты — он лишь хранит и предоставляет доступ к уже существующим сущностям.
